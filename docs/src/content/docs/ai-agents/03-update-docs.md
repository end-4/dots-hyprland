## The rule

**After making any code change, you must:**

1. Update the relevant **guide page** in `docs/src/content/docs/guide/`
2. Fill in the **AI Agent Notes** section of the matching **changelog entry**

This ensures the documentation stays accurate and other agents (or humans) can understand what you changed and why.

## Which guide page to update

| What you changed | Update this page |
|-----------------|------------------|
| Keybinds | `guide/03-usage.md` (keybind tables) |
| Autostart apps | `guide/04-configuration.md` (execs section) |
| Install dependencies | `guide/02-install.md` (packages section) |
| Shell UI / widgets | `guide/03-usage.md` (relevant feature section) |
| Quickshell services | `ai-agents/01-architecture.md` (services table) |
| Config options | `guide/04-configuration.md` |
| Bug fixes | `guide/05-troubleshooting.md` (if it's a common issue) |
| New features | `guide/01-overview.md` + relevant usage/config pages |

## Changelog entries

Changelog entries are auto-generated by the `docs-changelog.yml` GitHub Action on every push to `main`. Each entry is created at:

```
docs/src/content/docs/ai-agents/changelog/YYYY-MM-DD-<short-sha>.md
```

### Entry structure

```markdown
---
title: "<commit subject>"
date: YYYY-MM-DD
commit: <full SHA>
author: <committer>
---

## Changed Files

- `path/to/file1`
- `path/to/file2`

## Commit Message

```
<full commit message body>
```

## AI Agent Notes

<!-- AI agents: fill this in after working on related code.
     Explain WHY changes were made, any gotchas, and what
     other agents should know about this change. -->
```

### Filling in AI Agent Notes

When you find a changelog entry related to your work, add context:

```markdown
## AI Agent Notes

This change ported custom keybinds from `.conf` to `.lua` format as part
of the upstream Lua migration (PR #3269). The key translation was:

- `bind = MODS, KEY, dispatcher, arg` → `hl.bind("MODS", "KEY", hl.dsp.dispatcher("arg"))`
- `exec-once = cmd` → `hl.exec_cmd("cmd")` inside `hl.on("hyprland.start", ...)`

Note: `hl.dsp.togglespecialworkspace("")` requires an empty string argument,
not omission of the argument.
```

Good notes include:
- **Why** the change was made (not just what)
- **Gotchas** or non-obvious behavior
- **Related files** that might also need attention
- **Testing notes** — how you verified it works

## Running the docs site locally

```bash
cd docs
npm install
npm run sync    # Copies dots/custom/*.md into content/docs/custom/
npm run dev     # Starts dev server at localhost:4321
```

## Adding a new guide page

1. Create the `.md` file in the appropriate directory under `docs/src/content/docs/`
2. Add frontmatter:
   ```yaml
   ---
   title: "Your Page Title"
   description: "One-line summary"
   ---
   ```
3. Add the page to the sidebar in `docs/astro.config.mjs` if it's in `guide/` or `ai-agents/` (pages in `custom/` are auto-generated)
4. Use standard Markdown plus [Starlight components](https://starlight.astro.build/guides/components/)
